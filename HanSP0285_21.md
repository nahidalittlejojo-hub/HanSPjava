# <center><font face="楷体" size=6>继承</font></center>
## **一.基本介绍**
1. 基本语法
   ```
   class 子类 extends 父类{}
   ```
   + 子类会自动拥有父类定义的属性和方法
   + 父类又叫派生类或者超类或者基类
   + 子类又叫派生类
## **二.使用细节**
1. 子类继承了所有的属性和方法,但是私有属性不能在子类直接访问，要通过公共的方法去访问
   + private修饰的不可以访问
   + 默认的可以访问,如果子类和父类在同一个包里的话
   + protected,public父类不管是不是同包子类都可以访问父类
2. 子类必须调用父类的构造器,完成父类的初始化
   + 子类对象被创建时,先调用父类的构造器,在调用自己的构造器
   + 子类的构造器中会有系统默认的语句
      `super();`
      该语句会调用父类的无参构造函数
   + 当创建子类对象时,不管使用子类的哪个构造器,默认情况下总会去调用父类的无参构造器.
   + 如果父类没有提供任何无参构造器,则必须在子类的构造器中使用super去指定使用父类的哪个构造器完成对父类的初始化操作，否则error
3. 如果想要指定去调用父类的某个构造器,则显式的调用一下:
   `super(参数列表);`
   + super只能在构造器中使用,且在使用的时候,需要放在构造器的第一行
   + super()和this()都只能放在构造器的第一行,所以两个方法不能共存在一个构造器里
   + 注意如果不显式的写super,第一行写this是可以的
4. java所有类都是Object类的子类,Object是所有类的基类
   + 父类构造器的调用不限于直接父类,将一直往上追溯直到Object类(顶级父类)
5. 子类最多只能继承一个父类(指直接继承),即java中是单继承机制
6. 不能滥用继承,子类和父类之间必须满足is-a的逻辑关系
## **三.继承本质的分析**
1. 内存模型分析
   代码
   ```
   class Grandpa
   {
      String name="牛爷爷";
      String hobby ="旅游";
   }
   class Father extends Grandpa
   {
      String name =“小头爸爸”；
      int age =39;
   }
   class Son extends Father
   {
      String name ="大头儿子";
   }
   public class Test
   {
      main()
      {
         Son son = new Son();
      }
   }
   ```
   + 程序从main进入
   + 创建类对象前,会在方法区中先加载Object类的信息,在加载Grandpa类的信息,然后加载Father类的信息,最后加载Son类的信息
   + 各个类里形成继承关系
   + 首先,在堆中开辟一个空间,保存Grandpa类的信息.字符串在方法区的地址赋值给堆内存中的name和hobby
   + 其次，接着在Grandpa类信息的地址后面存储Father的信息,将方法区常量池中的字符串的地址赋值给Father类的name的空间中,int数据直接传给对内存空间
   + 然后,接着Father类存储信息的位置后面继续存放Son类的信息,将方法区的常量池中的字符串大头儿子的地址赋值给Son类的name变量空间中
   + 将存放一系列类信息的堆内存地址赋值给栈空间中的son变量的空间里.

2. 如何区分同一块堆内存中同名变量?如name
   + 要按照查找关系来返回信息
   + 首先,酰胺子类是否有该属性,如果子类有这个属性且可以访问，就返回该信息
   + 如果没有,就看父类有没有这个属性,如果有这个属性,且可以访问,就返回
   + 以此类推,持续向上级查找
   ```
      son.name;
      son.age;
      son.hobby;
   ```
   输出是 大头儿子 39 旅游
   + 注意:如果Father和Grandpa类中都有age变量,然而Father类中的age是private修饰,那么使用son.age访问age就会报错,直接截断,不会向上继续查找
